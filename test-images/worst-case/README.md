# Worst-case image fixtures

Input corpus for the print-pipeline integration test
(`scripts/test-pipeline.ts`). The goal is to prove that `normalizeInput` +
`convertToAdobeRgb` produce a valid print-ready file for any photo a
customer might realistically upload ‚Äî not just the clean, high-res happy
path.

## Layout

```
test-images/
‚îú‚îÄ‚îÄ worst-case/   ‚Üê input corpus (this folder)
‚îî‚îÄ‚îÄ output/       ‚Üê written by the test script, gitignored
```

The test script scans `worst-case/` for any `.jpg`, `.jpeg`, `.png`,
`.webp`, or `.heic` file and runs the full pipeline against each one.
There is no required filename convention ‚Äî drop in whatever you want
tested and re-run.

## Synthetic fixtures (committed)

These are generated by the test script on first run if missing, so a
fresh checkout has something to exercise without requiring binary blobs
in git:

| File | Why it matters |
|------|---------------|
| `synth-tiny-thumbnail.jpg`     | 480√ó640, forces upscale-factor reasoning |
| `synth-png-screenshot.png`     | PNG input path (no ICC, sRGB assumed) |
| `synth-stripped-metadata.jpg`  | No EXIF, no ICC ‚Äî checks we still embed sRGB |
| `synth-p3-tagged.jpg`          | Display-P3 ICC, tests color-space conversion |

Delete any of them to force regeneration. They are **not** representative
of real-world worst cases ‚Äî they exist so CI / fresh clones can run the
script without human intervention.

## Recommended real-world additions (drop in manually)

The synthetic set is intentionally thin. For a proper quality gate before
re-enabling large print formats, add the following photos from real
devices:

- **12 MP iPhone, P3 profile** ‚Äî a modern portrait in ambient light
- **12 MP iPhone HEIC** ‚Äî convert to JPEG on macOS before committing
  (`sips -s format jpeg input.HEIC --out output.jpg`)
- **Instagram screenshot** ‚Äî re-compressed JPEG, no ICC, low bit-depth
- **Mobile PNG screenshot** ‚Äî full-res phone screen capture
- **Facebook download** ‚Äî strippat profile, heavy JPEG compression
- **Motion-blur night shot** ‚Äî tests what sharp does with soft edges
- **Scanned physical photo** ‚Äî yellowed, with scanner dust, unusual gamut

Commit files < ~500 KB so the repo stays lean. Larger photos go in
`test-images/output/` (gitignored) and stay on your machine.

## How to run

```bash
npx tsx scripts/test-pipeline.ts
```

Prints per-file: input format, dimensions, output DPI at 30√ó40 / 50√ó70 /
70√ó100 targets, ICC profile in the resulting print file, pass/fail.

Exit code is non-zero if any file produces an invalid print file
(missing ICC, wrong chroma subsampling, < 150 DPI at 30√ó40).
